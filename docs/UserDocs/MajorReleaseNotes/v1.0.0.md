## Для пользователей

### Что умеет SubRadar

**Список подписок**
Все активные подписки в одном месте — с суммой, датой следующего списания и категорией. Видно сколько уходит в месяц и когда ждать очередного платежа.

**Календарь платежей**
Переключайся между видом по неделе, месяцу или году — смотри когда и сколько спишется. Удобно планировать бюджет наперёд.

**Уведомления**
SubRadar напомнит о списании за 1, 3 или 7 дней — настраивается под тебя. Никаких неожиданных трат.

**Три режима хранения данных**

| Режим | Для кого |
|---|---|
| 📱 **Локально** | Данные только на твоём iPhone, без облака и интернета |
| 🖥️ **Свой сервер** | Разворачиваешь бэкенд сам — полный контроль над данными, синхронизация между устройствами |
| ☁️ **Общий сервер** | В разработке — скоро будет |

**Переключение между режимами**
Сменил режим хранения — SubRadar спросит, перенести ли подписки в новое место или начать чисто.

**Тёмная и светлая тема**
Следует системной теме автоматически.

**Альтернативные иконки**
Несколько вариантов иконки приложения на выбор.

**Несколько валют и категорий**
Добавляй свои категории и валюты, группируй подписки как удобно.

## Для разработчиков

### Что вошло в релиз

**iOS (Swift + SwiftUI, iOS 17+)**
- Полный CRUD подписок с локальным хранилищем на SwiftData
- Кастомный таб-бар с календарём, добавлением и меню
- Коллапсирующий hero-заголовок с суммой расходов при скролле
- Pull-to-refresh списка подписок
- Подтверждение удаления (swipe и контекстное меню)
- Календарь платежей — неделя / месяц / год, генерация дат на лету из `nextBillingDate`
- Локальные уведомления через `UNCalendarNotificationTrigger`, настраиваемые lead times
- Онбординг с выбором режима хранения
- Экран подключения к self-hosted серверу с валидацией
- Миграция подписок при смене режима хранения
- Тёмная / светлая тема, альтернативные иконки приложения
- Поддержка нескольких валют и категорий
- Светлая и тёмная тема следуют системной настройке

**Бэкенд (Go + Chi, SQLite / PostgreSQL)**
- REST API: подписки, теги, категории, валюты
- JWT-авторизация (HS256, 30 дней)
- Self-hosted режим — вход по `server_secret`, один пользователь `admin@self-hosted.local`
- Email/пароль режим — для централизованного сервера (будущая монетизация)
- SQLite по умолчанию, PostgreSQL опционально
- Автоматические миграции через `golang-migrate` при старте
- Изоляция данных по `user_id` на уровне каждого SQL-запроса
- Единый статический бинарник, кросс-компиляция под linux/amd64, linux/arm64, darwin/arm64, darwin/amd64, windows/amd64
- Install-скрипт с автонастройкой systemd
- Update-скрипт с проверкой SHA256 и автоматическим применением миграций

### Архитектурные решения

- **Монорепозиторий** — iOS и бэкенд в одном репозитории
- **Offline-first** — iOS работает без сети, синхронизация при подключении
- **Протокол `StorageService`** — iOS не знает о конкретной реализации (Local / Remote), переключение через фабрику
- **Без Docker** — Go компилируется в единый бинарник, работает на Raspberry Pi
- **Даты в API** — RFC3339 без наносекунд (`time.RFC3339`), совместимо с iOS `JSONDecoder`

### Известные ограничения v1.0.0

- Общий сервер (`shared` режим) — UI есть, функциональность в разработке
- `RemoteStorageService` — авторизация и базовый CRUD работают, офлайн-очередь не реализована
- Уведомления не перепланируются автоматически после изменения подписки — требует захода в экран уведомлений
- Токен JWT не обновляется автоматически при истечении (30 дней) — требует повторного входа
