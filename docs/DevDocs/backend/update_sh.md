# update.sh — Документация

Скрипт обновления SubRadar Backend до последней версии. Корректно останавливает сервис, заменяет бинарник, применяет миграции БД и запускает сервис обратно.

---

## Быстрый старт

```bash
curl -fsSL https://raw.githubusercontent.com/OctopyApps/SubRadar/main/update.sh | sudo bash
```

---

## Требования

- SubRadar уже установлен через `install.sh`
- Linux (x86_64 или ARM64)
- Права root (`sudo`)
- `curl` и `sha256sum`

---

## Что делает скрипт

1. Проверяет что SubRadar установлен (бинарник есть в `/usr/local/bin/subradar`)
2. Получает текущую версию через `subradar --version`
3. Получает последнюю версию бэкенда с GitHub
4. Если версии совпадают — завершает работу, обновление не нужно
5. Скачивает новый бинарник и проверяет контрольную сумму SHA256
6. Останавливает systemd-сервис (если был запущен)
7. Заменяет бинарник
8. Применяет миграции БД
9. Запускает сервис обратно

---

## Важно про миграции

Миграции применяются **автоматически при старте сервера** — это встроенное поведение бэкенда. Скрипт запускает сервис после замены бинарника, и новые миграции применяются сами. Ручных действий с БД не требуется.

---

## Поведение при разных ситуациях

**Сервис был запущен до обновления** — скрипт остановит его, обновит, запустит снова. Даунтайм минимальный — только время замены бинарника.

**Сервис не был запущен** — скрипт обновит бинарник но не запустит сервис автоматически. Запустить вручную:
```bash
systemctl start subradar
```

**Уже последняя версия** — скрипт напишет об этом и завершится без изменений:
```
[SubRadar] У вас уже установлена последняя версия (v0.1.2)
```

**Контрольная сумма не совпала** — скрипт остановится с ошибкой и не заменит бинарник. Старая версия продолжит работать.

**Сервис не запустился после обновления** — скрипт выведет ошибку и подскажет как посмотреть логи.

---

## После обновления

Проверить что всё работает:

```bash
# Версия
subradar --version

# Статус сервиса
systemctl status subradar

# Логи если что-то пошло не так
journalctl -u subradar -n 50
```

---

## Частые вопросы

**Конфиг сбрасывается при обновлении?**

Нет. Скрипт меняет только бинарник — конфиг `/etc/subradar/config.yaml` и данные `/var/lib/subradar/` не затрагиваются.

**Можно откатиться на предыдущую версию?**

Да — скачайте нужный бинарник со [страницы релизов](https://github.com/OctopyApps/SubRadar/releases) и установите через `install.sh --local`:
```bash
sudo bash install.sh --local ./subradar-linux-amd64-v0.1.1
```

**Нужно ли обновлять iOS-приложение вместе с бэкендом?**

Зависит от типа релиза. Патчи (`backend/v0.1.x`) обратно совместимы — приложение обновлять не нужно. Минорные релизы (`v0.2.0`) затрагивают оба компонента — смотрите release notes.
